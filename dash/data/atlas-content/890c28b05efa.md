# Support — Not Forgotten Systems

How this install reports problems and gets them fixed. Bugs are monitored and repaired proactively by the consultancy's support desk; the owner is bothered only when judgment or money is involved.

## Severity tiers

| Tier | Meaning | Examples | Path |
| ---- | ------- | -------- | ---- |
| S1 | Agent down, security event, anything touching money | gateway dead, credential exposure, payment/refund errors | Instant alert via webhook + support desk acts immediately, owner's consultant alerted in parallel |
| S2 | A feature is broken | sends failing, cron dead, integration token expired, tool unreachable | Instant alert via webhook; support desk fixes solo, escalates only after two failed attempts |
| S3 | Cosmetic or minor | formatting quirks, slow responses, drift | Logged locally; picked up by the daily sweep; appears in the weekly digest only |

## How reporting works (three paths)

1. **Instant (S1/S2):** the on-error hook (`hooks/on-error.py`) fires the moment a tool or send fails: it classifies severity, writes a bug card, and POSTs a sanitized alert to the support relay.
2. **Liveness:** the support desk polls this machine's health endpoint every few minutes from outside. An agent that is down cannot self-report; the watchdog catches that.
3. **Daily sweep:** new bug cards in `support/outbox/` are pushed to the private client repo nightly; the support desk reconciles anything the instant path missed.

## Bug card format

One file per incident in `support/outbox/BUG-<timestamp>.md`, plus a one-line entry in `learnings/ERRORS.md`:

    ## [BUG-YYYYMMDD-HHMMSS] <component>: <short title>
    Severity: S1 | S2 | S3
    Status: open | fixing | fixed | escalated
    Detected: <ISO timestamp>
    Component: <tool / integration / agent>
    Symptom: <error text, sanitized>
    Attempts: <support desk appends one line per attempt>

## Sanitization rule (non-negotiable)

Bug cards and alerts carry symptoms, components, and error text only. Customer message bodies, names, emails, and phone numbers never leave this machine; the hook redacts emails and phone-shaped strings automatically, but do not rely on the redactor: never paste customer content into an error report.

## What the support desk may and may not touch

The support desk fixes code, configuration, skills, and cron. It NEVER modifies `Agent/Trust.md`, credentials, or the safety rules in `SOUL.md` without James Hardstaff's consultant in the loop. A fix is complete only after `verify.sh` and the relevant behavioural test pass again.
