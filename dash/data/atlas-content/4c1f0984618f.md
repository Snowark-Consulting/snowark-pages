# Connection Generation

**Purpose:** Surface non-obvious relationships between vault pages. Each run the vault gets slightly more interconnected.

> See [[wikilinks]] (Agent/wikilinks.md) for the dual-format convention, [[Consolidation SOP]] (Agent/consolidation-sop.md) for when this runs (step 4).

**Process:**
1. Review today's session activity and any new/modified vault files
2. For each key topic, person, system, or decision mentioned:
   - Search the vault for related pages that aren't already linked
   - Look for thematic connections (e.g. strategy discussion → business overview, person in session → people page)
3. Add cross-references using dual format: `[[PageName]] (path/to/file.md)`
4. Log connections made in the session notes under `## Connections Added`

**Useful connections:**
- System discussed in a session → its entry in knowledge/systems.md
- Person referenced → their entry in knowledge/people.md
- Strategic theme → knowledge/business.md
- Decision → affected knowledge page

**NOT useful:**
- Tenuous or speculative links
- Linking just because a keyword appears
- Circular references that add no information
