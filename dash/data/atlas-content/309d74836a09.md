# Vault Lint — Weekly Health Audit

**Schedule:** Weekly, Sunday 17:00 local time
**Purpose:** The honesty pass. [[Consolidation SOP]] (Agent/consolidation-sop.md) feeds the vault and auto-applies fixes to today's delta. Lint audits the whole corpus for rot that accumulates over time, and reports findings for review. It never auto-edits.

> See [[Infrastructure]] (Agent/Infrastructure.md) for the cron job config, [[wikilinks]] (Agent/wikilinks.md) for linking conventions audited in steps 1-3.

## Hard rules

- **READ-ONLY.** Never edit, rename, move, or delete a vault file. Output is a report only. The single exception is appending one line to `Agent/log.md` and writing the report file itself.
- **Mechanical first, judgment second.** Use `search_files`/`grep` to build candidate lists cheaply. Spend reasoning only on candidates, never on the whole vault.
- **Token-bounded.** Work from the wikilink graph, frontmatter, indexes, and the memory layer. Spot-read only suspected-problem files.
- **Precision over recall.** Only assert a contradiction you can cite with both sources. Do not flag speculative orphans.

## Scope

Scan `.md` files under the vault. Exclude: `venv/`, `node_modules/`, `cron/output/`, `logs/`, `state/`, `notes/` (transient), `sessions/`, `.sessions/`, `sandboxes/`, `lsp/`, `audio_cache/`, `image_cache/`, `__pycache__/`.

## Process

### 1. Build the link graph (mechanical)

- List in-scope `.md` files
- Extract every `[[wikilink]]` and every `(path/to/file.md)` reference
- Compute inbound-link count per file, and every reference whose target file does not exist

### 2. Orphans

Files with **zero inbound links**, excluding legitimate entry points: `SOUL.md`, `BOOT.md`, `Agent/Registry.md`, `memory/MEMORY.md`, `Agent/SOPs.md`, `Agent/Trust.md`, `identity/AGENT.md`, `Agent/Infrastructure.md`, knowledge index files, templates, and any `*README*`.

Report as "orphan candidates" — pages the graph cannot reach.

### 3. Broken / missing links

Wikilinks or paths pointing to files that do not exist (renamed, deleted, typo). Consolidation repairs today's; lint reports any that survived across the corpus.

### 4. Stale claims

Files or memory entries containing markers: `STALE`, `needs refresh`, `TBA`, `TODO`, `as of <old date>`, or `ARCHIVED` while still linked from an index.

### 5. Contradictions (judgment — candidates only)

Within the memory layer, and between memory ↔ knowledge pages, find conflicting facts. Only assert what you can cite with both sources and line references.

### 6. Index drift

- Files in `memory/` not listed in `MEMORY.md`; skills not in the `SOPs.md` index; significant docs absent from `Agent/Registry.md`
- Index entries pointing to files that have moved or been deleted

### 7. Gaps

Wikilinks referenced 2+ times with no target page — a concept the vault keeps naming but never defines.

### 8. Promotion candidates (structural growth)

Where the flat model is straining and a topic has earned its own home. Three shapes to flag (conservative — only fire on genuine cases):

- **Oversized:** any single file past ~200 lines. Recommend: split into dedicated home.
- **Dense cluster:** a group of 5+ related files that link heavily to each other but lightly to the rest of the vault. Recommend: promote to its own subdirectory with local index.
- **Disparate / scattered:** one initiative's material smeared across 3+ distinct locations with no single home. Recommend: consolidate into one location.

For each candidate report: the name, the evidence, and the recommended action.

## Output

1. Write a dated report to `Agent/lint-reports/YYYY-MM-DD-vault-lint.md`:
   - Header with counts per category
   - Each finding: file path, the issue, one-line recommended action. Grouped by category. Contradictions cite both sources.
   - A **"Recommended for next consolidation"** shortlist of the highest-value fixes
   - A **"Promotion candidates"** section if any exist (omit if none)

2. Append one line to `Agent/log.md`:
   `YYYY-MM-DD HH:MM | LINT | N findings (X orphans, Y stale, Z broken, W contradictions, P promotion candidates). Report: <path>`

3. Post a concise summary to the designated channel: heading, counts, the 3 highest-priority items, report path. Mobile-friendly — no tables, no code blocks. If the vault is clean, say so in one line.

## NOT to do

- **No auto-fixing.** Even trivial obvious breaks are only recommended — consolidation or manual review applies them.
- **No re-reporting resolved items.** Read last week's report in `Agent/lint-reports/`; if a prior finding is now fixed, note it resolved, do not re-raise.
- No tenuous contradictions, no speculative orphan flags.
