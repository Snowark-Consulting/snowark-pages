## Processing Rules

1. Read each unread email using the Gmail API (google_api.py)
2. For each email, craft a summary for Telegram:
   - **Sender name** — *Subject*
   - One sentence of what it is
   - Then a recommendation in one of these forms:
     - **From James:** "Would you like me to draft a reply / queue this in Airtable / start the follow-up sequence?"
     - **From anyone else:** "Would you like me to draft a reply / flag this for James / log this somewhere?"
   - If it's just test noise or FYI: "No action needed — just letting you know."
3. Never assume permission. Always ask. "Would you like me to..." is the standard.
4. After summarising, MARK AS READ immediately (remove UNREAD label)
5. Post summary to Telegram group with heading: **📬 Inbox — <count> new**

## Tone rules

- Proactive but permission-seeking. "Would you like me to..." not "I'll..."
- James's emails: offer specific, useful next actions (draft, queue, follow up)
- Everyone else: offer to draft a reply or escalate — never assume action
- One recommendation per email. Keep it tight.

## Posting to Telegram

After processing, send the summary via Telegram Bot API. Read the bot token from the environment:
```
export $(grep TELEGRAM_BOT_TOKEN /root/.hermes/.env | cut -d= -f2-)
```

Then post:
```
curl -s -X POST "https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage" \
  -d chat_id=-1003880645651 \
  -d parse_mode=Markdown \
  -d text="YOUR SUMMARY"
```

The agent's final response should be empty — the Telegram post IS the output.

## Mark as Read (CRITICAL)

After summarising each email, mark it read BEFORE posting the summary. This prevents the same email from appearing in the next poll. Use:
```
/root/.hermes/venv/bin/python /root/.hermes/skills/productivity/google-workspace/scripts/google_api.py gmail modify <MESSAGE_ID> --remove-labels UNREAD
```

## Gmail API reference

All commands via:
```
GAPI="/roo...  python3 /root/.hermes/skills/productivity/google-workspace/scripts/google_api.py"
```

Search: `$GAPI gmail search "is:unread" --max 10`
Get: `$GAPI gmail get <MESSAGE_ID>`
Mark read: `$GAPI gmail modify <MESSAGE_ID> --remove-labels UNREAD`

## Pitfalls

- `gog gmail mark read` (space) silently fails — always use `mark-read` (hyphen) with gog, or the Gmail API `modify --remove-labels UNREAD`
- `newer_than` only accepts d/m/y — never use for sub-day windows
- Always verify the UNREAD label is gone by re-checking labels after marking
