### Inbox pulse (LIVE — two-phase)

| Phase | Job | Cron ID | Schedule | Mode |
| ----- | --- | ------- | -------- | ---- |
| Phase 1 — unread scan | inbox-watchdog-phase1 | d0cdcda5b237 | */1 6-21 * * * Australia/Adelaide | no-agent (script-only) |
| Phase 2 — process + summarise | inbox-pulse-phase2 | 2d4a9ff4f06b | */1 6-21 * * * Australia/Adelaide | agent (full identity boot on mail found) |

Phase 1 runs `scripts/inbox-watchdog.sh` every 60s: checks Gmail API for unread, outputs JSON array of message IDs when mail found, silent when clean.

Phase 2 reads Phase 1's output via `context_from`. When empty: silent, no output. When IDs found: loads `Agent/pulse-bootstrap.md` → SOUL, Registry, MEMORY, LEARNINGS, gmail-card, email-int → reads each email, posts summary to Telegram via Bot API, marks as read. Deliver set to `local` so empty runs produce nothing visible. Telegram posting is explicit via curl when mail found.

Log: ~/.hermes/logs/inbox-pulse.log
