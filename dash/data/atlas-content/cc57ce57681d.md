# Secrets Pattern

One rule: secrets exist in exactly one place, the client instance.

- All credentials live in `~/.hermes/.env` (or service-specific env files), `chmod 600`, owned by the agent user.
- Config files reference env var NAMES, never values.
- The render script never touches secrets; `client.yaml` contains no credentials, only an inventory of WHICH integrations exist.
- Entering a credential happens in playbook 04, typed over `orgo ssh` or the orgo bash endpoint, never via a file that transits a repo.
- Rotation: each credential's expiry (if any) gets a cron reminder at creation time.
- If a secret ever lands in a repo or log: rotate it immediately, then fix the process and log the learning.
