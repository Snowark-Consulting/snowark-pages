# Integration: Meta Ads

## Prerequisites

- The business's Meta Business Manager account and ad account id; who holds admin
- Owner decision: read-only reporting (recommended at launch) vs the agent drafting campaign changes for approval

## Credential setup

- System user token in Business Manager, scoped to the one ad account, ads_read at launch (ads_management only when change-drafting is approved)
- TOKEN EXPIRY IS THE TRAP: record the expiry date the day the token is created and register a cron reminder two weeks before. An expired Meta token fails silently in scheduled jobs and the owner just stops getting reports
- Store per `core/security/secrets-pattern.md`

## Configuration

- Daily spend/results pull feeds the Analytics agent's scoreboard; Sales & Marketing agent flags anomalies (spend spike, CPL jump) same day
- All campaign changes are Tier 2: drafted, shown to the owner, applied only on approval

## Verification

- Pull yesterday's spend for the ad account; owner confirms it matches Ads Manager
- Confirm the token's expiry date and the renewal reminder cron both exist

## Known traps

- Token expiry (above). Treat every Meta token as a time bomb with a known fuse; the reminder cron is not optional
