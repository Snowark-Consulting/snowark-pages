# Wikilink Conventions

**Rule:** Every cross-reference between vault files must use the dual format: `[[PageName]] (path/to/file.md)`

**Why:** Obsidian uses `[[wikilinks]]` for graph view and backlinks. Agents use file paths for direct reads. The dual format serves both.

> See [[Connection Generation]] (Agent/connection-generation.md) for the rules on when to add links, [[Consolidation SOP]] (Agent/consolidation-sop.md) for when this is applied (step 3).

**When creating or editing any vault file:**
1. Reference another vault file → `[[PageName]] (path/to/file.md)`
2. Mention a person with a knowledge page → link them
3. Mention a system, product, or service → link it
4. Mention a concept with its own page → link it

**Do not over-link:**
- Don't link the same page multiple times in one paragraph
- Don't link common terms that don't have their own page
- First mention in a section is enough
