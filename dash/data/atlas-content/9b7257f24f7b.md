# Integration: WhatsApp

## Prerequisites

- Decide the path: WhatsApp Business API (via a provider such as Twilio or Meta Cloud API) for programmatic send/receive, or WhatsApp Business app on a dedicated number with the agent assisting via drafts. The API path requires a number NOT currently registered to a personal WhatsApp
- Owner's customer base actually uses WhatsApp (verify; in some trades SMS dominates)

## Credential setup

- Meta Cloud API: business verification on the Meta Business account, permanent token scoped to WhatsApp only
- Store per `core/security/secrets-pattern.md`; record scopes and expiry in the client repo

## Configuration

- Register webhook for inbound messages; route to the agent's channel handler
- Message templates: outside the 24-hour customer service window, only pre-approved templates can be sent; submit the needed templates (appointment confirmation, follow-up, invoice notice) for approval during setup, approval takes hours to days

## Verification

- Round-trip: owner messages the business number, agent acknowledges
- Template send: fire one approved template outside the session window, confirm delivery

## Known traps

- The 24-hour window rule: free-form replies are only allowed within 24h of the customer's last message; everything else must be an approved template. Build follow-up cadences around this or sends silently fail
