# Memory conventions

One file = one fact. Frontmatter:

    ---
    name: short-kebab-slug
    description: one-line summary used for recall relevance
    metadata:
      type: user | feedback | project | reference
    ---

Types: `user` (who the client is), `feedback` (corrections + confirmed approaches, with **Why** and **How to apply**), `project` (ongoing work, absolute dates only), `reference` (external pointers).

Rules: check for an existing file before creating a duplicate; update rather than fork; delete wrong memories; write terse (compressed shorthand, keep paths and specifics); convert relative dates to absolute; after writing, add one index line to MEMORY.md.
